Lucene Indexer Purpose
   This tool calls the methods to index the .html files and then create the graph structure.
 
How do I use this Page?
   If the user is using this without using Web Spider first, then the user must make sure the .html files are in the source_docs directory. Then the source_docs directory must be place in the jmapss directory in webapps.
   This tool will first index the .html files and create the .cfs files to be read into the graph structure. After the index is done then the graph structure is created. To view the nodes created use Graph Viewer to display them.
 
  1. If user uses Web Spider first then reference the Web Spider User Guide.


  2. If not using Web Spider.
    • Make sure a directory named source_docs is located in webapps/jmapss in tomcat. The source_docs directory should contain the .html files to be indexed.
    • The contents of the source_docs directory can contain directoryies or .html files.

  3. User Interaction.
    • There is no user interaction with Lucene Indexer after the link on the home page is clicked. When this page is brought up the code is indexing the .html files and then creating the graph structure.



  4. To see the nodes created reference the Graph Viewer User Guide.
